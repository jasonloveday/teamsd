Activity: CONFIRM Client Value and Experience
This activity includes the tasks used to ensure the client is satisfied with the solution and received the value expected.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose

The purpose of the CVM Confirm activity is to confirm that the client has received the expected value and is satisfied with the experience.  Starting with the client’s value drivers and expectations, the team confirms that the client:

  • is using the solution successfully to run the business,
  • realizing the value of the solution and the end-to-end experience with IBM,
  • is open to enhancing that value.

The progress indicator for this activity is a satisfied client who is willing to proceed with IBM on additional projects.  Experience with this project is used to update our understanding of the client (the Planning Phase) and to identify additional opportunities.

Description

At this point in the buying process, the client has implemented the solution and confirms the value received.

The following are done during this Activity by the relationship owner with involvement of those involved in the pre-sale solution design:

  • Confirm the client is using the solution to run the business.
  • Measure performance, impact and benefit.
  • Confirm that the client’s expectations are met and complete a transaction level survey.
  • Create plan to improve client satisfaction.
  • Increase exploitation new solution capabilities; explore ways to extend value.
  • Apply what we’ve learned to discover and explore new issues and “white space” opportunities.

We can verify our value based on :

  • Survey results and minutes from the Value Realization meeting confirm benefits and satisfaction.
  • Client agrees to be a reference.
  • Client engages IBM for follow on projects based on this projects success. 
  • Follow on opportunity / project is identified.

At the end of this Activity, the client's value is confirmed and the client is open to additional opportunities to work with IBM.

Key Considerations

Increasingly our clients may look to an iterative, agile approach to the delivery of value, with more of a series of projects rather than a single implementation project. Therefore, be specific on which aspect of Client Value was expected in a given delivery project and should be confirmed.
