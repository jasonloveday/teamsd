Phase: Pre-sale Solution Design
This phase is used to determine a client's requirements, the solution that will address them, and the corresponding decisions.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
The purpose of this phase is to gather the requirements, qualify the opportunity, architect and validate the solution, and close the sale.
Relationships
Parent Activities
  • Team Solution Design
Description

The primary work for each opportunity occurs here. The project is identified, defined and prepared for implementation/rollout. The key work products to be passed to the implementation team are created during this phase.

Key Considerations

Usage considerations for pre-sale (Team Solution Design): Pre-sale solution design is an iterative process, determining the solution from several points of view:

  • Contextual: why?
  • Conceptual: what?
  • Logical: how?
  • Physical: with what?
