Task: Monitor Pilot and Early Implementation
This task will evaluate customer satisfaction and value received during early implementation.
Purpose

It is most important for the Technical Solution Architect to be involved in early implementation if the client is to receive the value promised in the solution agreed to.  This includes:

  • Usually a pilot is performed before widespread deployment.  The scope of the pilot might be a subset of the users or a subset of the function.  
  • Education of implementers and users will occur during this task and should be evaluated for effectiveness. 
  • User acceptance is measured and rollout procedures/materials are evaluated.

Main Description

A pilot is optional, depending on the complexity of the solution and the amount of user and support training required.  If a pilot is to be done, there should be a written statement of the expected results and success criteria.   

A Project Management Review (PMR) provides a perfect opportunity for the pre-sale Technical Solution Architect to monitor early implementation.  If the project implementation has been broken into phases, a PMR might be held at the end of each phase.  Even if the project isn't being broken into time-boxed or functional "chunks", the project might include phases such as Solution Build, Solution Test, Pilot, Deployment, Run (Production) when PMR's might be held. 

There are likely to be lessons learned from the pilot and early implementation.  Project Management Reviews are a perfect opportunity for the Technical Solution Architect to understand these lessons and capture them for later projects. 

As a project progresses, there will be further elaboration of the work products, typically documented as Architectural Decisions.  There may have been changes to the requirements or scope which might have been documented as Change Cases (ARC 118).  Most of this experience is gained during this early phase of implementation. 

Key Considerations
Like other tasks in the Team Solution Design Activity: IMPLEMENT Client Solution, this Task is led by others in IGS and the client. The role of the Technical Solution Architect is to ensure promises made to the client are fulfilled and that lessons learned are acted upon. The only way to make this happen is for the solution designer to have at least an advisory role in Implementation.

If there is a teamroom or community used to store client information, the results of this task will be stored there for use on later projects.
