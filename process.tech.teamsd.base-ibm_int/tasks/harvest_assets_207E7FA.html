Task: Harvest Assets
This task will harvesting assets created in the engagement to produce a base of high quality reusable assets for future use
Disciplines: Asset Management
Purpose
Perform harvesting activity to produce a base of high quality reusable assets for future use.   Reuse considered should include client and IBM reuse.

      Main Description

      Determine which components should be selected to become reusable assets.  Reusable assets might include architectures, designs and code.   The pre-sale Technical Solution Architect is concerned with the identification of assets which can be used by this client in future engagements as well as selection of assets which can be reused within IBM for other clients.  This helps to ensure that the architecture and IBM products used in the project being deployed are exploited in other projects.   

      Assets to be used for this client are usually stored in a teamroom that has been set up for use by this client team. Before harvesting the results of the project, the Technical Solution Architect will need to update the work products to reflect what was actually implemented.  This might include lessons learned which can be captured as Architectural Decision as motivation, alternatives or implications of the decision.

      Although the Technical Solution Architect is primarily interested in harvesting the solution design assets, working with the implementation/delivery architect to capture the delivery assets will create the most valuable asset.  The most valuable assets also include guidance for re-use of the asset.  This is best captured when the project experience is fresh.

      Preparing the asset for use by people outside the immediate project team usually requires that the asset be updated to remove client sensitive information.  Once prepared, the best place to store it is iRAM, an internal instance of Rational Asset Manager sponsored by the Asset Architecture Board.  This asset library can be found at: https://w3-03.ibm.com/tools/cm/iram/  Further information on the AAB:  http://w3.blueprint.sby.ibm.com/b_dir/blueprint.nsf/All/30AD5D4C110185848525712C00021697?OpenDocument 

      The most valuable assets to be harvested from most client projects are reference implementations.  That is, assets describing the client project which are stored in iRAM as an "implementation" of one or more course-grained, public conforming assets.  Examples of these assets include reference architectures, selected Business Solutions, Service Products and SOA Service.  By defining this relationship of the client implementation to the high-value asset, you are making both assets more valuable.

      Key Considerations
      Like other tasks in the Team Solution Design Activity: IMPLEMENT Client Solution, this Task is led by others in IGS and/or the client. The role of the Technical Solution Architect is to ensure promises made to the client are fulfilled and that lessons learned are acted upon. The only way to make this happen is for the solution designer to have at least an advisory role in Implementation.
