Task: Explore New Client Issues
This task will evaluate additional client issues to understand the potential of solution extensions or related project opportunities.
Disciplines: Technical Requirements Development
Purpose
During this task, we'll look for ways to extend the capabilities of the new solution that was implemented to provide additional value. 
Relationships
RolesPrimary Performer:
  • Technical Solution Architect
Additional Performers:
Outputs
  • Client Imperative
  • Project Definition
  • Strategic Roadmap
  • Viability Assessment
    Process Usage
    • Team Solution Design  >  Support Implementation and Confirm Value  >  CONFIRM Client Value and Experience  >  Explore New Client Issues
    Main Description
    Grow use of new solution capabilities; explore ways to extend value.  Apply what we’ve learned to discover and explore new issues and “white space” opportunities.  Use the additional credibility that we gained with this project, by delivering what we promised, to find other business and IT challenges that we can address. This will lead back to the task of identifying new opportunities.
    Key Considerations

    Like other tasks in the Team Solution Design Activity: CONFIRM Client Value and Experience, this Task is led by others in IBM and the client. The role of the Technical Solution Architect is to ensure promises made to the client were fulfilled and resulted in the value expected. In looking for ways to extend that value, the Technical Solution Architect will play a pivotal role in understanding from where this additional value might come.

    If there is a teamroom or community used to store client information, the results of this task will be stored there for use on later projects.
