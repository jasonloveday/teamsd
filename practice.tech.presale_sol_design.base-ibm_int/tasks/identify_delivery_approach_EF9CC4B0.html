Task: Identify Delivery Approach
This task will develop a high level delivery plan in a pre-sale environment.
Disciplines: Schedule Management
Purpose
The purpose of this task is to define a high level delivery approach and preliminary delivery plan in a pre-sale environment in order to understand dependencies, cost, and/or risk drivers that will affect the price or winability of the solution.

        Main Description
        This task is to lay out the key the implementation and delivery aspects of IBMs offering in order to understand them, plan them, and accurately estimate them.  The implementation and delivery plan may include the further elaboration of the architecture that has been defined, integration of the solution into an existing environment, deployment across one or many client sites, and post installation activities such as support or maintenance.   The methods and artifacts necessary to execute the delivery aspects and additional risks identified will be documented in a form consistent with the size and complexity of the engagement.  This information will be taken into consideration when the effort is estimated.
        Steps
        Identify major delivery stages and responsibilities
        • Development, integration, test approach, deployment, installation, capability phasing, etc.
        • Identify perform organizations and their responsibilities for the various aspects of the delivery
        Identify special considerations
        • Client Preparation (e.g., facility modifications, training, organizational changes, etc)
        • Special environments (e.g., development environment, integration environment, test environment)
        • Data Security and Privacy Requirements: protection of IBM and client sensitive data from theft, loss, unauthorized access or disclosure and misuse. Note: GBS partners and Project Managers are accountable and responsible to ensure that these contractual Data Security and Privacy requirements are implemented and sustained on all client projects.
        • Specialty SME participation (Accessibility, safety, security and privacy, data migration, etc.)
        • Internal training of the IBM delivery teams (if using new technology, new tooling, unfamiliar new modules from a package provider, or some first of a kind implementations)
        Identify and tailor delivery method(s)
        • Select appropriate methods 
        • Identify the appropriate artifacts to be developed during delivery
        • Integrate artifacts and applicable methods steps into single project work breakdown structure. 
        • Use the solution design method adoption workshop approach if appropriate.
        Document the results
        •  This task is intended to be used with the Team Solution Design template: Project Definition for Team Solution Design.
