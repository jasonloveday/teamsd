Task: Identify High Level Data Sources
This task identifies high level data sources
Disciplines: Technical Architecture
Purpose
The purpose of this task is to identify and describe at a high level, data sources which are relevant to the proposed solution. This will contribute to an understanding of architectural aspects which are necessary to reuse existing data sources and an understanding of what additional data sources may be required. This task will also enable a more complete understanding of component responsibilities that include subject areas.

        Main Description

        This task documents in a graphical and/or textual format the major groupings of data that are needed to support the proposed solution. The resultant documentation may alternatively be referred to as a Conceptual Data Model or Business Information Model. The task is deliberately executed at a high-level. As such, the documentation is an excellent communication vehicle for an executive or business management audience. This task is generally developed in parallel to other tasks which document a high-level vision of a solution. While other tasks focus on the processes, organization or technology which needs to be developed or enhanced or purchased, this task focuses on the data which must be present in order to enable a capability or an application, i.e., it categorizes the multiple data sources. This task is the first iteration of tasks which more fully develop the model into a logical data model and physical data model. Industry and functional expertise is typically an important skill for the successful completion of this task.

        Steps
        Determine Scope of Relevant Data Sources
        Determine the basis and scope of the subject areas, related data items and data sources to be identified based on solution requirements and context.
        Identify Existing Data Sources
        Identify data existing sources that are candidates for use with the proposed solution.
        Analyze Existing Data Sources

        For each candidate data source, analyze the fit to the intended solution using considerations such as:

        • How does the data source satisfy the defined subject areas and required data items.
        • How the intended solution would use the data (create, read, update, delete) and whether those uses could be supported with existing data source
        • Who own or governs the data
        • What componentrs, mechanisms and protocols are available to access the data
        • Other existing data source data characteristics that intersect with non-functional requirements of the intended solution
        • What formats or data models are available that describe the data


        Identify Data Requirements Gaps

        Identify and document and data requirements gaps such as:

        • Data required by the intended application that is not available
        • Data conversion requirements
        • Data migration requirements
        • Information integration requirement
        • Other requirements gaps that will impact functional or non functional requirements and the resulting solution architecture
        • Requirements for components that would be responsible for additional data.
        Key Considerations

        Big Data and Analytics Opportunities require more detailed consideration and discovery of the information sources, which may be external as well as internal to the client's enterprise.  Consult the guidance for Big Data and Analytics for TeamSD for more detail. 

        Enterprise Architecture Influence: Existing enterprise architecture is an important influence on solution design. For additional guidance on this topic, see the associated Guideline: The Role of Enterprise Architecture in Solution Design. The following EA artifacts (or their equivalents) should be considered when performing this task.

        • Enterprise Information Model (ARC 307): The Enterprise Information Model will contains a high level subject area diagram and/or a high level ERD that provides input for developing the Subject Area Model for a particular solution.
