Work Product (Artifact): Zone Model (ART 0511)
This artifact provides a model of the zones into which the enterprise’s systems or an individual IT system is partitioned.
Purpose

This artifact provides a way of identifying areas of an IT system that share a common non-functional requirement and therefore need to be architected and designed to deliver a common service level characteristic. 

An IT system might be partitioned into zones associated with a particular level of security, availability, systems management, or performance.

This artifact provides a catalogue and description of the zones and their boundaries into which an enterprise’s systems or an individual IT system is partitioned.

Description
Main Description

This artifact models and describes at an architecture level the zones of an IT system and their characteristics and capabilities, and the boundaries between those zones.

This artifact’s terminology is defined within the System Description Standard R3 Semantic Specification (see Guidance for link) which includes definitions for:

  • Zones – an aggregation of model elements with a common (sub-) set of values for a particular non-functional requirement and/or non-functional characteristic.
  • Boundaries - associated with a change in value for a particular non-functional requirement and/or characteristic between two zones.
Brief Outline

Depending on the context, this artefact may consist of the following:

  • An introduction, describing the overall context and content of the artifact, the nature of the artifact (either to meet a specific set of IT requirement or to provide a generalized architecture pattern), its scope and depth of detail, its level, and its degree of completeness.
  • Diagram(s) showing the relative positioning of zones, the high-level characteristics of the zones, and the relationships between zones.
  • Diagram(s) mapping business and IT system elements to zones.
  • A list of zones.
  • A description of each zone’s characteristics and capabilities.
  • A list of boundaries.
  • A description of each boundary’s characteristics and capabilities.
Notation

This artifact is documented in a combination of structured text, structured tables, matrices and diagrams using a specialized notation and naming standards which are defined in the IBM System Description Standard V3 and the IBM Architecture Description Standard (ADS) Operational Model Notation Guidance V2.   Please refer to these for details of the notation.

Key Considerations

This artifact may be developed to different levels of detail and contain different content depending on the context in which it is being developed or used.

A zone represents an area or partition of a system which has a common set (or sub-set) of non-functional requirements.

Zones for different non-functional requirements may overlay each other and have different boundaries.

In some circumstances and for some non-functional requirements there may be a direct mapping between zones and locations.  In other circumstances, zones may be defined for “layers” of the IT system and therefore span multiple locations and multiple system elements.

In some circumstances, it may not be necessary to model the zones but it will still be important to describe the characteristics and capabilities of zones so that other artifacts and system elements reference a common set of zones.

Tailoring
Impact of not having

There will not be a clear understanding of nor an ability to analyse the areas of the system which have the same non-functional requirements and characteristics.

Reasons for not needing

This artifact may not be needed for simple IT systems with simple non-functional requirements and therefore do not need to be partitioned into different zones.

Representation Options

This artifact may be represented in any combination of the following; a drawing tool (e.g. Visio), a word processing tool (e.g. MS Word), and a spreadsheet tool (e.g MS Excel), and a database tool.
