Work Product (Artifact): Node (ART 0573)
This artifact describes a node or structured collection of nodes that represent, at an architecture level, the software and hardware that make up an IT system’s infrastructure.
Purpose

The purpose of this artifact is to provide a structured description or the software and hardware of an IT system at an architecture level. 

The purpose of this artifact varies depending on the context. For example,

  • Within the context of an Enterprise Architecture, this artifact describes the basic infrastructure building blocks and the placement of allowable components that are used to construct IT systems.
  • Within the context of the design and development of an IT system, this artifact is developed as part of the operational architecture aspect;
    • to support the placement of components,
    • to describe the overall physical configuration of the technologies and products necessary to deliver the functional and non-functional requirements of the IT system
    • to enable the evaluation and selection of IT infrastructure products.

Description
Main Description

This artifact describes nodes which represent the hardware (and software) that make up the IT system infrastructure.

A node is defined as a collection of components fulfilling a specific responsibility with a certain quality of service within the target IT system.

For an IT system, nodes are defined within the operational aspect of the system architecture and may be defined as part of an IT system or as part of a wider enterprise architecture.

Please refer to the IBM System Description Standard V3 for a definition of terms referred to by this artifact.

Depending on the development approach adopted and/or the context, it can be very helpful to create different levels of this artifact.  The two basic levels of this artifact are:

  • Logical:  This level describes the characteristics and capabilities of a node in a technology independent and product neutral manner.
  • Physical:  This level describes characteristics and capabilities of a node in a technology and product dependent manner.

Both levels of this artifact may be further described at three different sub-levels relating to the degree to which the level has been sized:

  • Unsized (or unranged) – the technology and products are specified to support the components deployed but not sized and without detailing how the system will achieve the service level characteristics.
  • Ranged – the technology and products are specified to support a bounded range of requirements for a defined range of circumstances.
  • Sized – the technology and products are specified and sized to support the components deployed and details how the system will achieve the service level characteristics.
Brief Outline

This artifact contains a structured description of a node or of collections of nodes.

Depending on the context, the level of the artifact, and the completeness of the artifact, this artifact contains:

  • A list of nodes
  • For each node:
    • Node name
    • Node description
    • A list of the deployment units (or components) placed on the node.
    • A description of the requirements and characteristics of the hardware and technology that the node represents derived from an aggregation of the deployment units (or components) placed on the node:
      • Hardware (CPU, memory, disk, and peripherals)
      • Communications
      • Systems Management
      • Non-functional requirements and service level characteristics such as performance performance, security (confidentiality, integrity and availability)
    • Optionally, an indicator as to whether the node is to be bought or developed.
    • Optionally, evaluation criteria to guide the selection of an IT platform for the node
  • A list of structured collections or groupings of nodes appropriate to the context
  • For each node collection
    • Node collection name
    • Node collection description
    • Node collection categorizations in terms of requirements and characteristics of the hardware and technology that the node collection represents.
Notation

The notation used is normally structured tables, structured text, or “node box” diagrams.  The notation is defined by the IBM Architecture Description Standard Notation for Operational Model V2 and the IBM System Description Standard V3.

Key Considerations

Please refer to the key considerations section of the Operational Model artifact.

Tailoring
Impact of not having

Without this artifact, there is no architectural level representation of the physical hardware and infrastructure of an IT system or as part of an enterprise architecture.  It therefore becomes hard to determine the requirements and service level characteristics that the hardware needs to support and deliver.

Reasons for not needing

This artifact may be unnecessary if the system is very trivial, e.g., a standalone PC application for a single computer, when it may be possible to omit architecture level artifacts and jump straight to design level artifacts which describe and specify the hardware and software configuration.

Representation Options

This artifact is normally represented by structured text, table or a node “box diagram”, uniquely naming and describing each Node. 

Whichever representation is selected, it should include a list of the deployment units placed on that node together with a description of the node’s non-functional characteristics.
