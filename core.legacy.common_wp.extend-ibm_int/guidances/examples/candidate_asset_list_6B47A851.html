Example: Candidate Asset List
The following is an example of a Candidate Asset List for a banking application.
Relationships
Related Elements
  • Candidate Asset List
Main Description
The example contains the following sections:
  • Requirements for Reuse: Areas in which assets should be searched. This includes both functional and non-functional requirements
  • Candidate Assets: List of all possible assets and the results of the assessment on whether the assets are applicable to the specific engagement
  • Asset Reuse/Creation plans (not added): For each selected asset, identify the development needs (tailoring, enhancement, creation).

The following example describes a subset of the project requirements. This list of requirements will evolve in each phase of development, as the system is understood in more detail.

Functional Requirements
This section describes the business function-related requirements that the assets must provide:

  • Bank customer presentation
  • Financial calculation (retirement planning, tax, budget)
  • Calendar
  • Reporting
  • Business graphics
Nonfunctional Requirements
This section describes system related requirements:
  • Persistence framework
  • National Language Support
  • Legacy system integration
  • Development metrics
