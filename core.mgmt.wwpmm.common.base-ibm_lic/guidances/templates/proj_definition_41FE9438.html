Template: Project Definition
This template provides a document used to describe the shape of the project (objectives and scope, stakeholders and proposed organization, major risks).

Main Description
Content Comment
Project identification  
Project / Sub-project  
Project / Sub-project manager  
Project objectives Based on the understanding of the project's objectives by the delivery organization, describe the set of objectives the project must achieve.
Project background  
Business needs and environment Description of the problem to be solved, and of the business areas under consideration. (Market segments, competitive analysis, sponsor activities etc.)
Project general scope and plan constraints Identify the specific needs to be addressed and provide key dates and states.
Solution background Provide a history of previous projects and solutions.
Key requirements List the high-level requirements that the project must meet.
Target solution and overall approach  
General presentation of the target solution Describe how the project will solve the business need.
Major components and level of uncertainty Provide a high-level breakdown of the major components of the solution. Include information about the level of uncertainty and complexity of the solution.
Key options for the technical environment If the project has special requirements regarding the technical environment then document those key options that influence cost, schedule, and possibly formal agreements when suppliers are involved.
Overall approach Describe, at a high level, the way in which the project will deliver the solution.
Project scope  
Position within the overall approach If necessary, position the project within the overall approach.
Scope Identify the boundaries of the project.
Key points Identify any key points that must be addressed in the formal agreement.
Major deliverables Outline the major products or services that will be delivered.
Customer satisfaction criteria (Conditions of satisfaction) Document any informal understanding of customer satisfaction issues that should be transferred, if needed, from the team who shaped the project (before the formal agreement) to the team who executes the project.
Planning framework  
High-level project schedule Provide a high-level project schedule for any projects or services and any sub-projects.
Brief description of each work unit Break the project down into its major work units. (High-level work breakdown structure)
Brief description of each sub-project If necessary, for each sub-project provide a brief description.
Key assumptions and rough estimates List the key assumptions which, if invalid, would make it necessary to revise the plan. Estimate roughly the amount of effort and duration of each work unit.
Key dates and dependencies List any key dates such as target dates for milestones. List any key dependencies.
Overall risks and risk response List any risk factors associated with the project. Identify planned responses to these.
Organization  
Project management Provide information about the structure that will manage the execution of the project.
Sponsor organization Provide information about the sponsor organization.
Delivery organization Provide information about the delivery organization.
Performing organizations Provide information about the performing organizations.
Initial organization breakdown structure (OBS) Description of how the project will be organized into project organizational units.
Necessary skills (high-level HR plan) Document any necessary skills that are required of project team members.
Initial project decision structure Provide an initial description of the Project Decision Structure.
Financial Costs, revenue and cash, as applicable.
Appendix A - Scope defining activities plan For scope defining activities, create a plan that includes:
  • Objectives
  • Organization
  • Work products
  • Milestones
  • Schedule
  • Management system
  • Assumptions, dependencies, and risks
  • Costs (if applicable)
Appendix B - Project planning activities plan For project planning activities, create a plan that includes:
  • Objectives
  • Organization
  • Work products
  • Milestones
  • Schedule
  • Management system
  • Assumptions, dependencies, and risks
  • Costs (if applicable)
Appendix C - Initial project definitions If necessary, for each sub-project, or project within a program, provide initial Project Definitions for each that includes:
  • Project name
  • Project description
  • Objectives
  • Key milestones
  • Major work units and associated work products
  • Start and finish dates
