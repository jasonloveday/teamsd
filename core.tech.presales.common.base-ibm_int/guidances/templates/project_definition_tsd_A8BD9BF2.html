Template: Project Definition for Team Solution Design
This template provides direction in a pre-sale context regarding the type of information required for the Project Definition Workshop work product.

Main Description

Team Solution Design users:  This template is to be used by the Technical Solution Architect using Team Solution Design for pre-sale Solution Design.  It is based on the template for ENG 343 but tailored for Team Solution Design pre-sale use. Some sections are left as-is. Sections that are changed to support the pre-sale context will include a note to explain the additional guidance.  These notes are always prefaced with, “Team Solution Design users:”. Some sections are not normally required in a pre-sale context and are indicated as such. In cases of IBM implementation, the Project Definition should be be conveyed to the implementation team for further elaboration.

Also attached are specialized templates to be used when the solution design is being led by Application Innovation Services (AIS) or Managed Business Process Services (MBPS) when using Team Solution Design.  This template extends the Team Solution Design template with additional guidance for AIS or MBPS.  In each case, this additional guidance or information to be supplied is marked as (AIS or MBPS). 

If the client has a defined format for some equivalent of Project Definition, you will need to use that format. Another example of a format required by the client would be a response to an RFP. Of course, you don't want to create and maintain information in two places, especially if it is changing quickly as the proposal is being pulled together. In this case, record in the Project Definition the information not included in the client 's document. And include in the Project Definition an appropriate reference to the client document for the information included there. Together, the Project Definition and the client document describe the project.
