Main Description

The first example (Visio file illustrating Customer Infrastructure) has a focus on what's installed at which location from a quantitative point of view.

The second example (file ilustrating Technical Environment Example) has more of a System Topology Diagram look and feel.

As these two examples show, a Technical Environment work product, including any diagram, might be very different depending on the purpose and the information available.  If the client has the information we need, we will usually just use it and not take time to reformat.
