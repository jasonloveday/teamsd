Main Description

This example illustrates a simple table that relates specific requirements to a particular software component that supports one or more requirements. For more complex situations, a spreadsheet with multiple tabs could be used to list requirements according to category such as functional, business, infrastructure etc. The table could also be constructed as an actual matrix, with requirements as shown and software components on the x-axis, using an x at the intersection to indicate a support relationship between a requirement and a component.
