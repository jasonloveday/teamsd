Main Description

In the context of IT solutions, patterns have long been available. The book Design Patterns  documents relatively fine grain patterns proven to be useful in program design. The book Patterns for e-Business  documents relatively large grain architectural patterns. A Wikipedia entry offers this definition of an architectural pattern in the context of IT solutions. “An architectural pattern is a general, reusable solution to a commonly occurring problem in software architecture within a given context.”  This guidance will illustrate brief examples of medium grain architectural pattern usage in the information domain of IT solutions. Examples are derived from the book Patterns of Information Management.
