Main Description

Once recognized, Non-functional Requirements are not particularly difficult to specify, at least not compared to the difficulty of meeting them.

The effort for this work product is proportional to:

  • Time taken to review available material
  • Number of stakeholders and IT staff to be interviewed to clarify NFR in available documentation. Allow 4 hours each person.
  • Number of NFRs that it is worth documenting

The effort required to complete this activity will depend on how much information is already available or can be easily gathered. For example, how much can be collected from an existing system /documentation, or how much will need to be collected ‘from first principles’ through meetings or workshops held with the customer.

In this latter case, effort can be reduced if the business analysts, modelers, etc. capture information relating to the non-functional requirements, e.g. volumetric, in parallel with the functional requirements. In this way we are able to capture the required NFR at the same time as we capture the functional aspects of the design, hence avoiding the need for additional NFR specific workshops.

Also, remember that this is not a one-off exercise. Time must be included to enable any assumptions made in documenting the NFRs to be validated with the customer over the course of the project.
