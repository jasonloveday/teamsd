Main Description

Lo-fi prototypes are pencil-and-paper renderings that are usually created during high-level design and typically depict screen content, layout, and flow. When demonstrated by a facilitator who “plays computer,” a lo-fi prototype can depict the behavior of the interface.

Hi-fi prototypes are usually written in a fourth-generation language (e.g., Visual Basic and VisualAge Java) and are usually created during late low-level design. A hi-fi prototype is a working model of the eventual software user interface. In many cases, such as Java and Web user interface prototypes, the accessibility considerations are also implemented/prototyped enough to be valuated. Hi-fi prototypes are highly interactive and are user-driven. Users can enter data in entry fields, respond to messages, select icons to open views, and, in general, interact with the user interface as though it were a real system or application.

Almost all aspects of a user interface can be prototyped, including user assistance. To create a lo-fi prototype (normally created on paper), follow these steps:

1. Assemble a prototyping kit consisting of paper, pencils, sticky notes, an eraser, labels, scissors, glue stick, index cards, and other similar items.
2. Review existing user data, i.e. user profiles, tasks, and working environment.
3. Select a use case for which to design a solution.
4. Brainstorm design ideas and alternatives.
5. Refer to design guidelines and other User Interface Architecture specifications, such as accessibility checklists (See Section 8 - References).
6. Sketch screens on paper. Use sticky notes as dialog boxes, message boxes, drop-down menus, and icons. Include descriptions of the visuals and any audio parts of the prototype.
7. Test designs and alternatives with users.
8. Iterate design during and after the testing session, based on user feedback.
9. Document screen widget behavior.
10. If this is a lo-fi prototype of the conceptual model, follow the 3x3 methodology, that is:

  • Identify and list key use cases.
  • Have users rate use cases according to importance, frequency, and satisfaction.
  • Have users identify and rate key attributes of a solution according to importance, frequency, and satisfaction.
  • Identify use case triggers, flow.
  • Brainstorm alternative models and metaphors, based on existing data.
  • Refine list to three.
  • Sketch three models three screens deep.
  • Test with users, emphasizing model and metaphor “fit.” Narrow to one.
  • Explore three alternative visual treatments of the chosen solution.
  • Test with users, emphasizing visual appeal, communication value.

11. To create a hi-fi prototype (show on the computer display), follow these steps:

  • Select a prototyping tool.
  • Review existing user data.
  • Refer to existing lo-fi prototypes.
  • Select two or more use cases to code.
  • Brainstorm additional design ideas and alternatives, visual treatments.
  • Refer to design guidelines and other User Interface Architecture Specifications, such as accessibility checklists (See Section 8 - References).
  • Code screens, widgets, etc.
  • Document prototype to provide more specific implementation details to code developers.

Note that there are variations of prototypes that fit in between lo-fi and hi-fi models. Computer drawing tools such as Paint, Paint Shop Pro, and Director can be effectively used to create a more realistic prototype.
