Example: Requirements Matrix for Inventory Management
Examples of business, general and technical requirements for inventory management

Main Description

Business Requirements

  • Ability to classify and track goods, materials/supplies by code, by property number, by cost center, by employee number, by location, by date of purchase, by warranty of equipment, by insurance code
  • Ability to receive electronic notification/report of orders on a daily basis w/detail available
  • Ability to see amt., qty, vendor information, cost center, delivery date
  • Ability to notify electronically the receipt confirmation to purchasing and pre-audit
  • Ability to track physical counts, condition, validate items, process damages, enter barcodes, approve P.O. for payment
  • Ability to set min/max for inventory items
  • Automatic notification in system when min. number is reached and generate a re-order flag in purchasing
  • Ability to capture signature of receiver electronically

General Requirements

  • Provide Robust Reporting & Analysis Capabilities
  • Ability for on-line inquiry throughout ERP system and ability to create inquiries
  • Provide the ability to batch run reports with unattended operation
  • Provide the ability to direct report output to either screen, printer or e-mail
  • Provide the ability to select the detail and scope for all reports
  • Allow for dynamic, data-sensing, color-coded exception reporting
  • Ability to route a copy of a screen print
  • Allow multiple users to access a single or many application modules simultaneously

Technical Requirements

  • Ability to operate on Microsoft Windows NT platform
  • Provide a standard User Systems Interface (i.e., GUI) throughout the application
  • Provide flexibility for application component distribution
  • Provide an automated software solution for the distribution and update of application software to all locations.  (For full releases and maintenance fixes and enhancements)
  • Provide the ability for users to define and maintain user preferences for windows, menus, hot keys, etc.
  • Provide capabilities in web-based technologies
  • Provides robust ad-hoc reporting facility and tools'
  • Allows for the creation of user defined formats for various on-going system generated reports
  • Provide a reporting facility that allows user defined distribution to various locations of selected reports
